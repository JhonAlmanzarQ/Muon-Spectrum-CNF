## Overview

Muons are secondary particles produced when cosmic rays interact with the Earth's atmosphere. Their distribution at ground level depends on the geophysical conditions of the site. The study of this component is relevant in applications such as muography, where muons are used to explore dense structures.

This project uses Conditional Normalizing Flows to generate the energy and angular spectra of the muonic component. ARTI/CORSIKA simulations are used as a reference to build a dataset across different sites, allowing the training of a model conditioned on geophysical variables such as altitude and the components of the geomagnetic field.

## Data

The project data are organized in the [`data/`](data/) folder.

This folder contains the raw simulations, the processed files, and the `lago_sites_from_jsonld.csv` file, which stores the geographic and geomagnetic information of the simulated sites.

## Notebooks

The `notebooks/` folder contains the main project notebooks.

These notebooks implement data preprocessing, hyperparameter search, robustness validation, and final model training.

## Source Code: `src/`

The `src/` folder contains auxiliary functions used by the notebooks.

These functions help organize repetitive parts of the code, such as processing, plot generation, model evaluation, and result handling.

## Workflow

To reproduce the main project workflow, the notebooks should be executed in the following order:

1. `00_preprocessing.ipynb`
   Performs the initial treatment of the ARTI/CORSIKA simulations, generates the processed files, and computes the estimated muon flux for each site.

2. `01_random-search-final.ipynb`
   Runs the random hyperparameter search to select candidate model configurations.

3. `02_robustness-validation-final.ipynb`
   Evaluates the robustness of the model through different splits or independent runs.

4. `03_model-final.ipynb`
   Trains the final model and evaluates it on the validation and test sets.

## Results

The results generated by the notebooks are stored in the [`results/`](results/) folder.

During the execution of the workflow, different outputs are generated, including:

- Processed files in `data/processed/`.
- Muon flux summary by site in `results/muon_flux.csv`.
- Muon flux versus altitude figure in `results/muon_flux_vs_height.png`.
- Result folders for hyperparameter search, robustness validation, and the final model.
- `.zip` files containing the complete results of each notebook.

## Model

The model corresponds to a Conditional Normalizing Flow trained to estimate the joint distribution of muon energy and zenith angle.

The model output variables are:

* Muon energy.
* Zenith angle.

The conditional variables used are:

- Site altitude.
- Horizontal component of the geomagnetic field.
- Vertical component of the geomagnetic field.

The goal is to generate muon spectra conditioned on the geophysical characteristics of the site, while maintaining consistency with the ARTI/CORSIKA simulations.

## Notes

- The project was developed mainly using Python notebooks. The use of a GPU is recommended for model training.
- It is recommended to adjust the paths inside the notebooks according to the execution environment used.
- To train and evaluate only the final model, the processed data must first be generated with `00_preprocessing.ipynb`, followed by the execution of `03_model-final.ipynb`.
- For a more interactive exploration of the results, the project web platform can be accessed at: https://muon-generator.vercel.app/simulador
