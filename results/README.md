# Results

This folder is intended to store the results generated by the preprocessing, training, validation, and final evaluation notebooks.

## Structure

```text
results/
├── muon_flux.csv
├── muon_flux_vs_height.png
├── Random_search/
├── Random_search.zip
├── Robustness_validation/
├── Robustness_validation.zip
├── Model_Final/
├── Model_Final.zip
└── README.md
```

## Preprocessing outputs

The notebook `00_preprocessing.ipynb` generates two main outputs in this folder.

### `muon_flux.csv`

This file contains a summary of the estimated muon flux for each simulated site.

It is generated from the processed simulation files and includes the information required to analyze the dependence of the muon flux on the altitude of each site. The expected content includes site identifiers, altitude values, event counts, and estimated flux values.

### `muon_flux_vs_height.png`

This figure shows the relationship between the estimated muon flux and the altitude of the simulated sites. The plot includes a Poisson-based exponential fit used to describe the altitude dependence of the muon flux.

## Notebook result folders

The notebooks `01_random-search-final.ipynb`, `02_robustness-validation-final.ipynb`, and `03_model-final.ipynb` generate result folders inside `results/`.

These folders are also compressed as `.zip` files to make the outputs easier to download, archive, or share.

The generated result folders are:

```text
results/Random_search/
results/Robustness_validation/
results/Model_Final/
```

and their compressed versions are:

```text
results/Random_search.zip
results/Robustness_validation.zip
results/Model_Final.zip
```

Depending on the notebook, these folders may include:

- trained model weights;
- model configuration files;
- training histories;
- normalization statistics;
- validation and test metrics;
- summary files;
- error reports;
- energy spectrum plots;
- angular spectrum plots;
- NLL training curves;
- train, validation, and test site splits.

## Notes

- The files in this folder are generated automatically by the notebooks.
- Large output files, trained models, and compressed result folders are excluded from the repository.
- The compressed `.zip` files are useful for downloading or archiving the complete results of each notebook execution.
- To reproduce these results, run the notebooks in order, starting with `00_preprocessing.ipynb`.
