# Data

This folder contains the data used in the preprocessing of the ARTI/CORSIKA simulations and in the construction of the dataset used by the muon spectrum estimation model.

The information is organized into two main folders: `raw/`, for the raw data obtained from the simulations, and `processed/`, for the data generated after the initial data treatment. In addition, this folder includes the file `lago_sites_from_jsonld.csv`, which contains the geographic and geomagnetic information of the simulated sites.

The `processed/` folder stores the files generated after the treatment of the raw data.

During this process, the original simulations are converted into `.csv` files, preserving the variables required for the muon flux analysis and model training.

## Sites file: `lago_sites_from_jsonld.csv`

The `lago_sites_from_jsonld.csv` file includes example sites used in this project; however, it can be replaced with another file with the same structure when using new simulation sites.

File columns:

| Column | Description |
|---|---|
| `sigla` | Simulation site identifier |
| `lat` | Site latitude |
| `lon` | Site longitude |
| `h` | Altitude above sea level, in meters |
| `bx` | Horizontal component of the geomagnetic field |
| `bz` | Vertical component of the geomagnetic field |
| `bi` | Total geomagnetic field intensity |

File example:

```csv
sigla,lat,lon,h,bx,bz,bi
cha_3600,-16.3500,-68.1300,5230.0000,22.3800,-4.6200,23.1120
ata_3600,-23.0058,-67.7592,5105.0000,20.6380,-8.5980,22.5000
khun_3600,36.510,75.250,4700.000,28.896,44.161,52.726
```

The `sigla` column must match the name used to identify each processed simulation.

## Notes

- The raw ARTI/CORSIKA data are not included in the repository due to their size.
- The processed files can be generated by running the `00_preprocessing.ipynb` notebook.
- The `lago_sites_from_jsonld.csv` file should be included in the repository, as it contains the metadata required to reproduce the data treatment.
- The `raw/` and `processed/` folders are kept in the repository using `.gitkeep` files.
